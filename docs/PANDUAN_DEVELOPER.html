Panduan Developer TutorQA

Panduan teknis bagi tim pengembang untuk menjalankan, memahami arsitektur, dan mengelola deployment sistem tanya jawab belajar mandiri berbasis model bahasa mT5-small.

Persyaratan Sistem

Sebelum menjalankan aplikasi TutorQA di server lokal atau server produksi, pastikan sistem Anda telah memenuhi kriteria berikut:

  • Python 3.10 atau versi di atasnya (sangat disarankan untuk kompatibilitas tokenizer HuggingFace).
  • Direktori Model Terlatih: Folder mt5-qa-indonesia/ wajib terisi lengkap dengan berkas parameter model (termasuk model.safetensors, config.json, dan tokenizer.json).
  • Sistem Operasi: Kompatibel dengan Windows, macOS, dan Linux.

Langkah Instalasi

Pasang seluruh modul dependensi pustaka yang dibutuhkan menggunakan pip package manager melalui shell terminal Anda:

# 1. Masuk ke direktori proyek utama
cd "/Punya GW/01. CODE/NLP"

# 2. Pasang semua dependensi dari requirements.txt
pip install -r requirements.txt

Menjalankan Server Lokal

Jalankan server aplikasi web menggunakan modul python run script bawaan. Modul ini meluncurkan Uvicorn ASGI server dengan pengaturan hot-reload otomatis:

python run.py

Setelah server aktif, buka peramban Anda dan akses tautan berikut:

👉 Tautan Lokal: http://localhost:8000

⚠️ Catatan Penting: Hindari membuka alamat IP internal host 0.0.0.0 pada kolom peramban karena beberapa peramban modern memblokir CORS/akses kamera atau modul Javascript tertentu dari domain tersebut.

Daftar API Utama (REST Endpoints)

Backend FastAPI menyediakan tiga antarmuka API utama berbasis JSON untuk memproses materi dan menjawab pertanyaan:

Endpoint Metode Format Input Tugas Utama
/api/upload POST multipart/form-data Menerima dokumen PDF pelajaran, mengurai isinya, menyimpan teks ke memori, dan memberikan ID dokumen unik beserta saran pertanyaan awal.
/api/ask POST application/json Menerima ID dokumen dan pertanyaan teks. Menemukan paragraf relevan, menjalankan model mT5 untuk mengekstrak span jawaban, dan memformulasikan paragraf jawaban kronologis.
/api/suggestions POST application/json Mengekstrak kata kunci konseptual penting dari dokumen atau input teks manual pengguna dan menyusun saran pertanyaan cerdas secara instan.

Mengatasi Masalah Port (Port 8000 Sedang Digunakan)

Jika Anda menemui galat port sedang digunakan (port in use) ketika meluncurkan server lokal, jalankan perintah terminal berikut untuk mematikan proses pengganggu:

# 1. Cari ID Proses (PID) yang mengunci port 8000
netstat -ano | findstr :8000

# 2. Hentikan paksa proses pengunci port menggunakan PID hasil pencarian
taskkill /PID <MASUKKAN_PID_DI_SINI> /F

Sebagai alternatif, Anda juga dapat membuka berkas run.py di root dan menyesuaikan parameter port (misalnya mengubah ke port 8080).
